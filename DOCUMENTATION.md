## What You Are Building

You are working on a Zendesk Help Center theme.

This repository contains:

1. Theme layout and templates.
1. Styling and behavior files.
1. GitLab pipeline to validate, back up, deploy, and roll back.

Project layout is intentionally separated:

1. Runtime theme files stay at repository root (required by Zendesk import):
   - `manifest.json`, `script.js`, `style.css`, `templates/`, `assets/`, `settings/`, `translations/`
1. Supporting automation files are under `tooling/`:
   - `tooling/scripts/` for deployment, validation, versioning, and rollback scripts
1. Developer editor tasks stay in `.vscode/` for VS Code integration.

Source control is GitLab only.
Deployment is done from GitLab to Zendesk using Zendesk CLI (ZCLI).

---

## Basic Terms You Must Know

1. `Branch`: Your personal work copy in Git.
1. `Merge Request (MR)`: A request to merge your branch changes.
1. `Pipeline`: Automated jobs in GitLab.
1. `Artifact`: File generated by a job and stored by GitLab.
1. `Production`: Live Zendesk theme used by real users.
1. `Backup`: ZIP copy of current production theme before deploy.
1. `Rollback`: Revert production using a backup ZIP.
1. `ZCLI`: Zendesk command-line tool used to preview/import themes.

## Phase 1: First-Time Setup (Onboarding)

### Step 1: Install Required Tools

Install Node.js and npm.

On macOS with Homebrew:

```bash
brew install node
```

Install Zendesk CLI:

```bash
npm install -g @zendesk/zcli
```

Verify installation:

```bash
node --version
npm --version
zcli --version
```

### Step 2: Clone Project

```bash
cd ~/Desktop
git clone https://git.hilti.com/BU_FPS/sw-support-group/FPS_ZD_SKC_PE.git
cd FPS_ZD_SKC_PE
```

Check files are present:

```bash
ls
```

You should see files like `manifest.json`, `style.css`, `script.js`, `templates`, `assets`, `.gitlab-ci.yml`.

### Step 3: Authenticate ZCLI

```bash
zcli login -i
zcli themes:list
```

If `themes:list` fails, run login again.

### Step 4: Optional VS Code Task Usage

From Command Palette (`Cmd+Shift+P`):

1. Run `Tasks: Run Task`.
1. Choose one of these tasks based on your goal:
   - `Zendesk: Login` for first-time authentication.
   - `Zendesk: Preview Theme` for local preview while editing.
   - `Zendesk: List Themes` to view existing Zendesk themes.
   - `Zendesk: Deploy Any Branch (Interactive)` to pick branch, target brand, and deploy with confirmation prompts (runs `tooling/scripts/deploy-any-branch-interactive.sh`).
   - `Zendesk: Deploy Current Branch (Non-Interactive)` to deploy the current checked-out branch directly using the default brand ID (runs `tooling/scripts/deploy-current-branch-noninteractive.sh`).

Note for new developers:

1. Use `Zendesk: Deploy Any Branch (Interactive)` if you are not sure which branch or brand to deploy.
1. Use `Zendesk: Deploy Current Branch (Non-Interactive)` only when you are sure you are on the correct branch and target brand.

## Phase 4: CI/CD and Production Deployment

The current pipeline stages are:

1. `validate`
1. `release`
1. `backup`
1. `deploy`

Current jobs:

1. `theme_version_dry_run` (automatic)
1. `theme_validate_structure` (automatic)
1. `theme_version_release` (manual, default branch only)
1. `theme_backup_production` (manual, any branch)
1. `theme_deploy` (manual, any branch)
1. `theme_rollback_production` (manual, any branch)

### Required GitLab CI/CD Variables

Set these in GitLab: `Settings > CI/CD > Variables`

1. `ZD_SUBDOMAIN`
1. `ZD_EMAIL`
1. `ZD_API_TOKEN`
1. `ZD_THEME_ID`
1. `DEPLOY_CONFIRM_REQUIRED` with value `DEPLOY_TO_PROD`
1. `ROLLBACK_CONFIRM_REQUIRED` with value `ROLLBACK_TO_PROD`
1. `ZCLI_VERSION` (example: `1.0.0-beta.56`)

Security rules:

1. Mark `ZD_EMAIL` and `ZD_API_TOKEN` as masked and protected.
1. Never commit secrets to repository files.

### Deployment Safety Rules

Deployment is blocked if:

1. Confirmation variable is missing or wrong.
1. Backup artifact does not exist in pipeline.
1. Required Zendesk variables are missing.

### VS Code Deployment Tasks (Recommended for Junior Developers)

The repository includes two deployment tasks in `.vscode/tasks.json`:

1. `Zendesk: Deploy Any Branch (Interactive)`
   - Best for guided deployment.
   - Lets you select branch and brand interactively.
   - Asks for final deployment confirmation.

1. `Zendesk: Deploy Current Branch (Non-Interactive)`
   - Best for repeated deployment to the same target.
   - Uses the currently checked-out branch.
   - Uses default brand ID `36275984782609` unless `ZD_BRAND_ID` is set.
   - Fails if uncommitted changes exist.

Example overriding brand ID before running the non-interactive task:

```bash
export ZD_BRAND_ID=36275984782609
```

## Detailed Example: Deploy Current Branch from GitLab

This example uses the current branch:

`FPSKB-201-auth-ui-toggle`

### Step 1: Push the Branch

```bash
git checkout FPSKB-201-auth-ui-toggle
git push origin FPSKB-201-auth-ui-toggle
```

### Step 2: Open Branch Pipeline in GitLab

1. Go to `CI/CD > Pipelines`.
1. Open the pipeline for `FPSKB-201-auth-ui-toggle`.
1. Confirm both validate jobs passed:
   - `theme_version_dry_run`
   - `theme_validate_structure`

### Step 3: Run Backup Job (Mandatory)

1. In the same pipeline, click `theme_backup_production`.
1. Click `Run`.
1. Wait for status `Passed`.
1. Confirm artifact exists under `backups/`.

### Step 4: Run Deploy Job with Confirmation

1. Click `theme_deploy` in the same pipeline.
1. Click `Run`.
1. Add variable when starting job:

`DEPLOY_CONFIRM=DEPLOY_TO_PROD`

1. Start the job.
1. Wait for `Passed`.

### Step 5: Verify Deployment

1. Open Zendesk Help Center.
1. Check the updated page/feature.
1. If issue found, execute rollback section below.

## Rollback Procedure (If Deployment Fails)

### Fast Rollback from Backup Artifact

1. Open the same pipeline used for deployment.
1. Ensure `theme_backup_production` had passed.
1. Run job `theme_rollback_production`.
1. Provide variable:

`ROLLBACK_CONFIRM=ROLLBACK_TO_PROD`

1. Optional: if you want a specific backup ZIP, provide:

`ROLLBACK_BACKUP_FILE=backups/theme-<theme-id>-backup-<timestamp>.zip`

If optional variable is not provided, the newest backup ZIP is used automatically.

### What Rollback Job Does

1. Finds backup ZIP artifact.
1. Extracts it.
1. Imports the extracted theme to Zendesk using ZCLI.
1. Restores production to previous state.

## Troubleshooting (Non-DNS)

### Error: Unauthorized During Backup or Deploy

Cause:

1. Wrong `ZD_EMAIL`.
1. Wrong or expired `ZD_API_TOKEN`.

Fix:

1. Verify CI/CD variables.
1. Regenerate token in Zendesk if needed.
1. Retry job.

### Error: Theme Not Found During Backup

Cause:

1. Wrong `ZD_THEME_ID`.

Fix:

1. Get correct theme ID from Zendesk Guide admin.
1. Update variable.
1. Re-run job.

### Error: Deploy Blocked by Confirmation

Cause:

1. `DEPLOY_CONFIRM` not set.
1. Value does not match required token.

Fix:

1. Run deploy with:

`DEPLOY_CONFIRM=DEPLOY_TO_PROD`

### Error: Rollback Blocked by Confirmation

Fix:

1. Run rollback with:

`ROLLBACK_CONFIRM=ROLLBACK_TO_PROD`

### Error: Backup Artifact Not Found

Fix:

1. Run `theme_backup_production` in same pipeline.
1. Wait until passed.
1. Run deploy again.
